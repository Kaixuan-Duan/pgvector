# 用户界面描述

本项目是pgvector PostgreSQL插件的增强版本，在保留原有单向量列HNSW索引和IVFFlat索引的全部功能基础上，新增双向量列HNSW索引存储与检索能力、多ORDER BY词典序排序检索方法、基于RRF算法的多路向量融合排序方法，以及与查询优化器的深度集成功能。

相比原生pgvector，本项目的核心增强体现在以下四个方面。第一，双向量列HNSW索引允许用户为同一张表的两个向量列创建单一联合索引，使用CREATE INDEX ON table USING hydex (col1 vector_l2_ops, col2 vector_l2_ops)语法即可完成，系统自动为每列独立构建HNSW图结构，并通过元数据页统一管理两列的维度、距离函数和图层次信息。第二，多ORDER BY词典序排序检索支持用户在单个查询中按优先级依次指定两列的距离条件，例如ORDER BY col1 <-> query1, col2 <-> query2，系统利用双向量列索引逐层获取最近邻结果，仅在需要时计算次列距离。第三，RRF融合排序通过rrf函数实现，调用格式为rrf(col1, op1, q1, col2, op2, q2, k, w1, w2, cand1, cand2)，该函数接收两列向量、距离算子对象标识符、查询向量及可选参数，默认平滑常数60、权重各0.5、候选集各200，返回融合分数供ORDER BY排序使用。第四，查询优化器自动识别包含RRF函数的排序语句，在规划阶段将执行路径重写为基于双向量列索引的CustomScan，用户无需手动指定Hint即可获得索引加速的融合排序性能。

在向量类型支持方面，本项目针对双向量列索引和RRF融合排序功能，支持vector和sparsevec两种类型的自由组合。用户可为第一列选择vector类型存储稠密向量，为第二列选择sparsevec类型存储稀疏向量，两列可分别使用不同的距离算子进行检索。所有类型转换和距离计算均通过扩展内置的运算符完成，用户界面层保持与原生PostgreSQL一致。

综上所述，本项目通过标准SQL接口，在完全兼容pgvector原有功能的前提下，提供了双向量列索引、词典序排序检索、RRF融合排序和优化器自动重写四项核心能力。用户仅需掌握标准CREATE INDEX和SELECT语法，配合rrf函数调用，即可实现多路向量检索结果的高效融合排序。
