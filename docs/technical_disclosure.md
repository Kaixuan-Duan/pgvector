# 技术交底书

## 提案概述

本专利提供PostgreSQL双向量列HNSW索引及RRF融合排序技术方案，支持单一索引管理两列向量数据，通过词典序排序、RRF流式批处理与理论剪枝、查询优化器Hook集成四项核心技术，实现多路向量检索结果的高效融合排序。

## 应用场景/应用对象

适用于电商多模态搜索、推荐系统多行为召回、知识库稠密稀疏向量联合检索、金融风控多维异常检测等需要将两路向量检索结果进行融合排序的PostgreSQL数据库应用场景。

## 关键词

**双向量列索引**：指在数据库中为同一张表的两个向量列创建单一联合索引的技术，通过统一的索引结构同时管理两列向量的元数据和图结构，支持基于两列的联合近邻检索，区别于传统单一向量列索引。同义词/扩展词：multi-column vector index、双列向量索引、混合向量索引。

**HNSW**：层次可导航小世界图，是一种基于多层图结构的近似最近邻搜索算法，通过构建类似导航树的层次化图结构，在高维向量空间中快速定位最近邻，广泛应用于向量数据库的索引构建。同义词/扩展词：hierarchical navigable small world、近似最近邻图索引。

**RRF**：倒数排序融合，是一种多路检索结果融合算法，通过将不同检索路径的结果按排序位置倒数加权求和，生成统一的融合分数，常用于多模态或多特征检索场景中的结果整合。同义词/扩展词：Reciprocal Rank Fusion、多路检索结果融合。

**查询优化器Hook**：PostgreSQL数据库提供的扩展机制，允许在查询规划阶段插入自定义逻辑，拦截并修改执行计划，本专利中用于识别RRF查询并重写为CustomScan路径以实现索引加速。同义词/扩展词：planner hook、执行计划重写、CustomScan路径定制、PostgreSQL查询拦截。

**向量检索**：基于向量空间模型的相似性搜索技术，通过计算查询向量与数据向量的距离来寻找最近邻，广泛应用于语义搜索、推荐系统和多模态检索等领域。同义词/扩展词：vector search、ANN近似最近邻搜索、embedding相似性检索、语义检索。
