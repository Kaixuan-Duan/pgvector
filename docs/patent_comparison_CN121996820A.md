# 与CN121996820A的对比分析

## 相同点

两者均面向多向量对象检索场景，针对同一对象具有多个语义空间向量表示的情况提供索引加速方案。两者均采用图索引结构组织向量数据，通过邻居关系构建实现近似最近邻搜索，以降低高维向量检索的计算复杂度。两者均在索引构建阶段考虑多向量特性，将多向量信息融入索引结构而非独立建索引后外部融合。两者均关注查询阶段的效率优化，通过特定的搜索策略减少不必要的距离计算和索引遍历。

## 不同点

CN121996820A构建的是统一多空间图索引，将所有向量组合空间的邻居关系整合到单一图结构中，查询时根据向量组合和权重执行自适应导航搜索，未涉及关系型数据库集成。本专利是PostgreSQL数据库扩展，通过双向量列HNSW索引在数据库层管理两列独立图结构，提供词典序排序和RRF融合排序两种SQL查询接口，并通过查询优化器Hook自动重写执行计划为CustomScan路径，实现数据库原生多路向量检索融合，无需外部索引服务。
