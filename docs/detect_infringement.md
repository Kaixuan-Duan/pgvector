# 如何发现他人产品采用本发明

对于发明点1双向量列HNSW索引存储结构，可通过分析数据库索引的元数据页格式进行检查。具体而言，可以创建包含两个向量列的测试表并建立索引，随后使用十六进制编辑器或数据库调试工具读取索引文件的元数据页。若元数据页中存在能够同时存储两列维度信息、两列距离函数标识以及两列独立图层次结构的字段布局，而非仅支持单列信息的传统HNSW元数据格式，则可判定采用了本发明的双向量列索引存储架构。

对于发明点2多ORDER BY词典序排序方法，可通过监控查询执行过程中的距离计算次数进行检查。具体而言，构造一个两列向量的查询语句，使用第一列排序并限制返回条数，同时监控第二列距离函数的调用次数。若在多数情况下第二列距离的计算次数显著少于第一列，呈现按需计算的惰性求值特征，而非对两列均进行全量距离计算后再排序，则可判定采用了本发明的按需计算次列距离的优化方法。

对于发明点3基于流式批处理与理论上限剪枝的RRF算法，可通过注入测试数据并监控扫描深度进行检查。具体而言，构造两列查询使得其中一列的权重明显高于另一列，执行RRF融合排序查询并监控每一轮批处理后的扫描深度。若系统在满足Top-N结果的最低分数阈值高于理论上界时提前终止扫描，且监控日志中出现阈值与上界的比较信息以及提前终止的记录，则可判定采用了本发明的理论上限剪枝机制。

对于发明点4查询优化器集成框架，可通过分析查询执行计划进行检查。具体而言，执行包含RRF函数的ORDER BY查询，使用数据库的EXPLAIN命令查看执行计划。若计划中显示VectorRRF CustomScan节点而非普通的SeqScan或IndexScan配合Sort节点，且rrf函数在执行器层面被替换为列引用而非逐行函数调用，则可判定采用了本发明的优化器Hook拦截与CustomScan路径重写机制。
