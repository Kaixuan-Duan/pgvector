# 与基于向量相似度与SQL结构化约束的混合检索专利对比分析

## 相同点

两者均涉及向量检索与关系型数据库的结合，目标是在数据库环境中实现高效的相似性搜索。两者均采用双存储架构设计，将向量数据与传统结构化数据分开管理并建立关联。两者均接收包含向量查询条件和结构化约束的检索请求，并协同执行多类检索任务。两者均关注检索结果的融合与排序输出，以提升查询的精准度和效率。两者均有扩展SQL语法或使用标准SQL接口降低用户使用门槛的设计思路。

## 不同点

该对比专利侧重于向量语义匹配与SQL结构化筛选的深度融合，核心在于将向量相似度检索与传统SQL条件过滤结合，解决的是单一向量列的语义检索加结构化筛选问题。本专利聚焦于双向量列的检索结果融合排序，核心在于为同一对象的两列独立向量构建联合HNSW索引，并通过RRF算法或词典序排序实现多路向量检索结果的融合，解决的是两列向量的联合近邻搜索问题。该对比专利采用向量存储空间与关系型存储空间的双存储架构，本专利则在PostgreSQL单一数据库内通过扩展机制实现，利用CustomScan深度集成查询优化器。该对比专利扩展SQL语法以支持向量检索与结构化约束的混合查询，本专利则完全基于标准SQL的ORDER BY和函数调用语法，通过优化器Hook自动识别并重写执行计划，对用户透明。
