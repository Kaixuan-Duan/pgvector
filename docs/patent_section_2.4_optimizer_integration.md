# 2.4 PostgreSQL双向量列索引的查询优化器集成框架的实现方法

本发明提供了一种PostgreSQL双向量列索引的查询优化器集成框架的实现方法，该方法在查询规划阶段通过Hook机制识别包含RRF函数的查询语义，将其执行计划从逐行函数计算模式重写为基于双向量列HNSW索引的CustomScan路径，并通过投影列表改写与执行器协同机制，在查询语义不变的前提下大幅提升执行效率。

需要特别说明的是，RRF函数（另定义于第3发明点）的实现为一个桩函数：当处于CustomScan执行上下文时，通过全局指针直接返回预计算的RRF分数；当未处于该上下文时，则报错提示必须与ORDER BY子句联用以启用CustomScan优化。这种双重设计使得RRF函数既作为SQL层面的语义标记供优化器识别，又作为执行器层面的分数通道供上层节点引用，从而将声明式查询接口与底层索引加速执行解耦。

本发明的核心在于查询规划阶段的Hook拦截与路径重写机制。在扩展初始化时，VectorRrfInit函数注册自定义扫描方法并安装set_rel_pathlist_hook钩子，将vector_rrf_set_rel_pathlist_hook函数挂载到PostgreSQL的路径生成阶段。当优化器处理基表关系时，该Hook函数首先调用原有的Hook链，然后检查当前关系是否为简单基表关系。若是，则检查查询的排序子句是否包含且仅包含一个排序键，并通过find_rrf_sort_expr函数从排序子句对应的投影列表条目中查找RRF函数表达式。该函数通过strip_expr_wrappers剥离可能的类型包装节点，检查是否为函数表达式节点，并确认排序方向为降序。若未找到RRF函数或排序方向不符，则直接返回不做任何干预。

找到RRF函数表达式后，Hook函数提取其参数列表，要求至少包含六个参数，即两列的向量、算子和查询向量。前两个参数emb1和emb2必须为变量节点，表示表中的列，且两个列必须来自同一张表。随后调用pick_index_for_vars函数在表的索引列表中查找同时包含这两个列的双向量列HNSW索引，通过匹配索引键与列属性号确定列在索引中的位置。若未找到匹配的索引，则返回不做处理。

确认索引后，Hook函数提取剩余可选参数，包括平滑常数k、权重w1和w2、候选集大小cand1和cand2，若未提供则使用默认值。同时从查询的limitCount中提取LIMIT表达式。然后构建CustomPath路径节点，设置路径类型为自定义扫描，父节点为当前关系，启动代价为0，总代价为1以确保优化器优先选择该路径，路径排序键为查询的排序路径键以表明该路径已提供排序结果避免额外的排序操作。CustomPath的custom_private字段按固定顺序保存索引对象标识符、两列位置、LIMIT表达式、两个算子表达式、两个查询向量表达式以及可选参数表达式，共计13个元素。

在路径转计划阶段，vector_rrf_plan_custom_path函数将CustomPath转换为CustomScan计划节点。该函数首先创建CustomScan节点并继承custom_private参数列表，然后调用build_custom_scan_tlist_for_table函数构建自定义扫描的投影列表。该函数打开基表关系，遍历所有属性列创建对应的变量节点作为投影条目，并在最后追加一个双精度浮点常量作为RRF分数的占位列，其资源编号为属性列数加1。随后调用replace_rrf_in_tlist函数遍历原始投影列表，通过表达式树变异器将其中所有RRF函数调用替换为指向分数占位列的变量节点。这样，上层的投影操作在引用RRF分数时将直接读取CustomScan预计算的值，而非调用函数。

在执行阶段，CustomScan的执行方法由vector_rrf_begin、vector_rrf_exec等函数实现。vector_rrf_begin函数从CustomScan的custom_private中提取索引对象标识符和两列位置，打开索引关系，初始化所有参数表达式状态，创建内存上下文和堆元组槽，并调用vector_rrf_prepare_results函数预计算RRF结果集。vector_rrf_exec函数按顺序遍历预计算的结果集，通过索引获取接口从堆表中读取可见版本的元组，将堆列值和RRF分数填入扫描槽，执行过滤条件检查后返回结果元组。每行结果的RRF分数通过全局指针current_rrf_score传递给SQL层面的rrf桩函数，使得当上层节点调用rrf函数时直接返回该分数而非重新计算。扫描结束时清空全局指针避免污染后续查询。

综上所述，本发明通过查询优化器Hook在规划阶段识别RRF查询语义，将其重写为基于双向量列HNSW索引的CustomScan路径，避免了逐行函数调用的开销；通过投影列表重写将RRF函数引用替换为预计算分数的列引用，保持了SQL语义的透明性；通过全局指针机制实现执行器与SQL函数之间的分数传递，使得上层节点无需感知底层优化。整个方法在PostgreSQL的查询优化器和执行器框架内无缝集成，既保留了标准SQL的调用接口，又实现了索引加速的高效执行。
